## [0.4.1] - 2025-10-31

### üîß Refactor Minimale Failsafe - Sistema Auto-Riparante

**Obiettivo:** Rendere il sistema di slot validation robusto, auto-riparante, e debuggabile in produzione.

**Problema:** L'errore `fp_exp_slot_invalid` persisteva in produzione senza possibilit√† di debug (WP_DEBUG=false, log vuoti).

### Changed

- **üî¥ CRITICO - Logging Sempre Attivo**: `ensure_slot_for_occurrence()` e checkout ora loggano SEMPRE (non pi√π condizionato a WP_DEBUG)
  - **Prima**: `if (defined('WP_DEBUG') && WP_DEBUG) error_log(...)`

  - **Beneficio**: Debug possibile in produzione tramite `/wp-content/debug.log`
  - (`src/Booking/Slots.php`, `src/Booking/Checkout.php`)

- **üü° Signature Change - ensure_slot_for_occurrence()**: Ora ritorna `int|WP_Error` invece di solo `int`

- **‚úÖ WP_Error Dettagliati**: Tutti i failure point ora ritornano WP_Error con dati completi
  - Include: `experience_id`, `requested_start`, `requested_end`, `buffer_before`, `buffer_after`, `conflicting_slots`
  - Visibili nei log per diagnosi immediata
  - (`src/Booking/Slots.php`, `src/Booking/Checkout.php`)

- **‚úÖ Endpoint Diagnostico**: Nuovo endpoint `/diagnostic/checkout` per debugging avanzato
  - Mostra: carrello, availability meta, simula slot creation
  - Accessibile solo da admin con permission `can_manage_fp()`
  - (`src/Api/RestRoutes.php`)

- **‚úÖ Conflicting Slots nel Log**: Quando c'√® buffer conflict, logga i primi 5 slot in conflitto
  - Query dettagliata per mostrare QUALE slot causa il conflitto
  - Include ID, start_datetime, end_datetime

### Fixed

- **üêõ Impossibile Debuggare Checkout in Produzione**: WP_DEBUG=false impediva qualsiasi logging
  - **Soluzione**: Logging sempre attivo, indipendente da WP_DEBUG
  - **Impatto**: Debug ora possibile in produzione tramite `debug.log`

- **üêõ Capacity=0 Blocca Checkout**: Esperienza con `slot_capacity=0` causava `fp_exp_slot_invalid`

- **üêõ Errori Generici Senza Dettagli**: Messaggio "slot non disponibile" senza contesto
  - **Soluzione**: WP_Error con tutti i dati (experience_id, datetime, buffer, conflicting_slots)
  - **Impatto**: Diagnosi immediata invece di guesswork

### Developer Notes

### Fixed

- **üî¥ CRITICO - Checkout Slot Validation Fallisce con Capacity=0**: Risolto triplo problema critico nel checkout:
